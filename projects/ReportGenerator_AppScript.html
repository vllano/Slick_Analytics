Slides Report Generator

Project Overview

The Report Generator project addresses the challenge of time-consuming and error-prone manual report generation for technical test data. The project aims to significantly reduce the time and effort required for operators to create post-collection reports. By automating data extraction, report population, and file linking, the Report Generator streamlines the reporting process, minimizes user input errors, and ensures report consistency. This automation allows operators to allocate more time to data collection and other critical post-collection tasks. The solution utilizes Google Workspace and Apps Script to generate Google Slides reports with a few clicks, based on data from testing apps and other sources.

Problem Statement

The Field Operations team faced significant challenges with their report generation process. My analysis revealed the following key issues:

  • Inefficiency: Manual report creation was a time-consuming process, diverting resources from core testing activities.
  • Inconsistency: Reports varied in format and content, making it difficult to compare data across different tests.
  • Data Accuracy: Manual data entry from device logs to reports was prone to human error, potentially compromising report reliability.
  • Scalability: The manual process was not scalable to handle the increasing volume of test data.
  • Template Management: The use of multiple report templates added complexity and increased the risk of using the wrong template.

Requirements and Solution Design

To address these challenges, I defined the following key requirements for the automated solution:

  • Automation: Automate the extraction of data from testing apps and population of Google Slides templates.
  • Standardization: Ensure consistent report formatting and content across all tests.
  • Accuracy: Minimize manual data entry to improve data accuracy and reduce errors.
  • Efficiency: Reduce the time required to generate reports.
  • Template Management: Automate the selection of the correct report template.
  • Scalability: The solution should be able to handle a growing volume of test data.

The proposed solution, the Report Generator, was designed to meet these requirements by:

  • Automating data retrieval from testing apps using unique identifier keys.
  • Dynamically selecting the appropriate report template based on test type and device configuration.
  • Automatically generating charts and tables to ensure data accuracy and consistency.
  • Saving generated reports to a designated Google Drive folder for easy access and organization.

Implementation and Data Flow

The Report Generator was implemented using Google Apps Script, leveraging the following Google Workspace services:

  • Google Sheets: Device performance data from testing applications was exported to Google Sheets using BigQuery data export functionality.
  • Google Forms: Used to gather supplementary file uploads and relevant links.
  • Google Slides: Used to create and populate report templates.
  • Google Drive: Used for storage and organization of report templates and generated reports.

The automated report generation process follows these steps:

  • Data Input: The script reads test data and report parameters from a specified Google Sheets spreadsheet.
  • Template Selection: The script automatically selects the correct Google Slides template based on pre-defined criteria.
  • Report Generation: The script creates a copy of the template and populates it with the relevant data from Google Sheets.
  • Output: The generated report is saved as a Google Slides presentation in a designated Google Drive folder.

Results and Business Impact

The implementation of the Report Generator has resulted in significant improvements:

  • Increased Efficiency: Report generation time has been reduced by automating the process, freeing up operator time for other tasks.
  • Improved Data Accuracy: Automation has minimized manual data entry errors, leading to more reliable and accurate reports.
  • Enhanced Consistency: Standardized report templates ensure a consistent format and presentation of data.
  • Increased Scalability: The automated process can handle a larger volume of reports, supporting future growth in testing activities.
  • Reduced Turnaround Time: Faster report generation enables quicker dissemination of test results to stakeholders.
